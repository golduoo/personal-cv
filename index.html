项目 1：Modular RAG MCP Server（个人项目） | 2026-01 至今

Problem：面向 Agent 工具链落地场景，解决 RAG 组件接入成本高、协议不统一、跨平台复用困难的问题。

Data：使用多源文档库（文本与图片）构建知识检索数据集，并设计 5 条标准化测试用例用于离线评测。

Approach：基于 MCP 协议将检索能力封装为标准 Tool，支持 Claude Desktop / Cursor 直接调用；检索层采用 Dense + BM25 双路召回、RRF 融合与 Cross-Encoder 重排；构建多模态 RAG Pipeline（Vision LLM 图像语义描述 + 图文混合返回）；设计 Provider 抽象层（Factory + Strategy）以 YAML 一键切换 OpenAI / DeepSeek / Ollama / Azure 与不同 VectorStore 后端；基于 Ragas + Streamlit 搭建自动化评测与可视化 Dashboard。

Outcome/Impact：在 5 条测试用例上，混合检索 Hit Rate 达 100%，MRR 达 0.52（top-10）；实现可插拔、可评测、可观测的模块化 RAG 中间件，显著降低 Agent 侧接入与迭代成本。

Your Contribution：独立完成架构设计、检索策略实现、多模态链路开发、评测体系搭建与可视化交付。

Tech Stack：Python · MCP · ChromaDB · BM25 · LangChain · Ragas · Streamlit

项目 2：基于 CLIP + ESE + FAISS 的图文跨模态检索系统（个人项目） | 2026-02 至 2026-03

Problem：针对跨模态检索中精度与效率难以兼顾的问题，探索低维嵌入下高质量图文匹配方案。

Data：基于 Flickr8k 数据集，使用冻结 CLIP (ViT-B/32) 提取图文特征，并构建图文检索评测集。

Approach：复现 ICLR 2025 ESE（Espresso Sentence Embeddings）中的 Learn-to-Express 与 Learn-to-Compress 双机制；在冻结 CLIP 特征上训练约 2.1M 参数轻量投影网络；将 512 维嵌入压缩至 128 维，并基于 FAISS 构建 Flat 精确索引与 IVF 近似索引。

Outcome/Impact：嵌入压缩后存储与检索开销降低 75%，平均 Recall 保持原始性能 91%（74.72% vs 81.68%）；在同为 128 维条件下，相比直接截断，Text-to-Image R@5 提升 33%，Image-to-Text R@1 提升 28%；支持毫秒级大规模图文检索。

Your Contribution：独立完成论文复现、训练流程实现、压缩对齐实验设计与 FAISS 检索系统搭建。

Tech Stack：PyTorch · OpenCLIP · FAISS
